# AGENTS.md

## Mission

Refactor the EBS Tech website into an extremely modern, futuristic, premium, and visually impressive digital experience.

The website must make visitors instantly feel that EBS Tech is not a generic software agency, but a **high-end technology company** operating at the intersection of:

- software engineering
- artificial intelligence
- martech
- intelligent automation
- digital growth systems

The experience must feel bold, premium, cutting-edge, and product-driven.

The redesign must create a strong **"wow effect"** when users land on the website.

---

## Core Objective

When someone enters the website, they must think:

- "This company is advanced."
- "This feels like the future."
- "These people build serious products."
- "This is not a normal agency."
- "These guys understand AI, growth, product, and software deeply."

The website must communicate **innovation through execution**, not through clichés.

## Technical Expectations

When refactoring the site, the agent must:

1. keep the codebase maintainable
2. preserve performance
3. improve structure before overpolishing
4. centralize design tokens where possible
5. standardize surfaces, spacing, radii, shadows, and gradients
6. refactor progressively and safely
7. avoid unnecessary breaking changes
8. make the design system scalable

If styling is fragmented, create a more coherent system.

If components are repeated, abstract them.

If the page feels flat, increase visual depth intelligently.

---

## What the Agent Must Not Do

Do not:
- make the site look generic
- make the site look like a template
- create a plain institutional software website
- overuse effects until readability suffers
- create weak or obvious agency copy
- hide the martech / AI positioning
- treat Imóvel Certo as a minor note
- create a boring portfolio section
- build visually flat sections
- use inconsistent iconography
- make dark mode low contrast and hard to read
- use too much pink or too many accent colors
- produce a site that feels common
